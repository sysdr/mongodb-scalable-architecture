Day 8 — MongoDB Scalable Architecture

Content Feed & Product Catalog

Compound-indexed collections for efficient queries

About this project

  • Content feed — Posts by user and publish time; indexed by userId + publishTime for fast “get feed by user” and time-range queries.
  • Product catalog — Products by category and ID; indexed by categoryId + productId for fast category browsing and lookups.
  • Both collections live in contentDB. This dashboard shows live document counts and server operations.
  • Use Run demo to execute sample queries and see metrics update.

Live metrics

Actions

Run demo to send queries and refresh metrics. Run start to (re)start MongoDB and seed data. Run tests to validate setup.

Application workflow

App → Feed by user & time → contentDB → Index (userId, publishTime) → Results
App → Products by category → contentDB → Index (categoryId, productId) → Results

Data flows from the application through indexed queries into MongoDB and back. Compound indexes keep feed and catalog queries fast.
